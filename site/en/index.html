Why it helps

Not a generic site prompt — a delivery workflow

This skill constrains the agent to read source materials first and separate facts from unknowns before moving into blueprint, benchmark research, design, implementation, and QA. It will not invent customers, data, credentials, awards, or form-submission results.

01

Evidence first

Separate confirmed facts, missing information, forbidden assumptions, and the asset inventory — reducing false claims on public pages.

02

Taste gate

For premium or public projects, study external references first, extract design logic and a divergence contract, and avoid a generic template feel.

03

Auditable

Built-in static audit, SEO/GEO, screenshot-level visual review, and delivery-grade placeholder interception, so teams share one quality bar.

Visual system discovery

Choose a visual system before writing the page

The workflow has the agent propose 2-3 named visual systems before premium, public, or multi-demo projects. Each system defines layout grammar, signature element, proof treatment, motion signature, and mobile behavior.

Editorial Evidence Chroma

Evidence color ribbons

For brand, creative, and public skill pages. Replace plain cards with colored evidence ribbons and editorial layouts.

Quick start

Get your team running in 3 minutes

After cloning into the agent's skills directory, invoke it directly with `$business-website-skill`. An install script writes to common directories too.

Install for Codex 
git clone https://github.com/ChuluuMGL/business-website-skill.git
cd business-website-skill
scripts/install.sh codex
Build a lightweight package 
python3 scripts/package_runtime_skill.py
# dist/business-website-skill-runtime.zip
QA before delivery 
python3 scripts/audit_static_site.py <site-root> index.html \
  --strict-seo --no-placeholders

Quality gate

Make delivery checks the default action

Teams don't just check whether a page looks good — they check fact provenance, mobile, links, images, SEO/GEO metadata, and unresolved placeholders.

Static audit

Checks anchors, assets, duplicate IDs, viewport, image alt, and CSS asset references.

SEO/GEO ready

Checks title, description, canonical, Open Graph, Twitter Card, JSON-LD, and the main H1.

Taste & screenshot review

Public or client-delivery pages require benchmark notes, differentiation, module boundaries, blank-zone anomalies, and mobile screenshots.

Runtime matrix

Publicly records verification status for Codex, Claude Code, Cursor, Trae, OpenClaw, Hermes, and more.

  6. 5BuildStatic site, React/Vite, Next, or existing stack
