Why it helps

不是普通建站提示词，而是一套交付流程

这个 Skill 约束 Agent 先读资料、区分事实和未知项，再进入蓝图、标杆研究、设计、实现和 QA。它不会编造客户、数据、资质、奖项或表单提交结果。

01

证据优先

把已确认事实、缺失信息、禁止编造项和素材清单分开，减少公开页面里的虚假表述。

02

品位闸门

高级或公开项目先研究外部参考，提炼设计逻辑和差异化契约，避免同质化模板感。

03

可验收

内置静态审计、SEO/GEO、截图级视觉检查和交付级占位内容拦截，方便团队统一质量标准。

Quick start

3 分钟让团队用起来

克隆到 Agent 的 skills 目录后，直接用 `$business-website-skill` 调用。也可以用安装脚本写入常见目录。

安装到 Codex 
git clone https://github.com/ChuluuMGL/business-website-skill.git
cd business-website-skill
scripts/install.sh codex
生成轻量包 
python3 scripts/package_runtime_skill.py
# dist/business-website-skill-runtime.zip
交付前验收 
python3 scripts/audit_static_site.py <site-root> index.html \
  --strict-seo --no-placeholders

Quality gate

让交付检查变成默认动作

团队使用时，不只看页面是否好看，还要检查事实来源、移动端、链接、图片、SEO/GEO 元数据和未处理占位内容。

静态审计

检查锚点、资源、重复 ID、viewport、图片 alt、CSS 资源引用。

SEO/GEO 就绪

检查标题、描述、canonical、Open Graph、Twitter Card、JSON-LD 和主 H1。

品位与截图检查

公开或客户交付页面需要检查标杆记录、差异化、模块边界、空白异常和移动端截图。

运行时矩阵

公开记录 Codex、Claude Code、Cursor、Trae、OpenClaw、Hermes 等运行时验证状态。

Workflow

从资料到交付的 7 个阶段

  1. 0问诊网站类型、技术栈、视觉方向
  2. 1证据地图事实、未知项、禁止编造项
  3. 2网站蓝图站点地图、区块、CTA、证明模块
  4. 3标杆与设计参考研究、品位闸门、视觉路线
  5. 4实施计划文件、组件、素材、交互、命令
  6. 5构建静态站、React/Vite、Next 或现有栈
  7. 6-7QA 与交付验证结果、预览方式、未确认事项
