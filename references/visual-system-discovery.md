# Visual System Discovery

Use this reference when a website needs to feel distinctive, premium, public-facing, or meaningfully different from other demos. This step comes after the evidence map and site blueprint, and before implementation.

The goal is to make the agent choose a complete visual system, not only a color preset. A strong business website should have a named point of view, a layout grammar, a signature element, a proof/evidence treatment, a motion signature, and a mobile behavior.

## Core Principle

For high-taste work, do not ask the user to choose from vague labels such as "premium", "tech", or "modern". Offer 2 to 3 named visual systems that describe how the website will actually be built.

Each system must answer:

- What business context is it for?
- What should the visitor feel or believe in the first 5 seconds?
- What is the layout grammar?
- What is the signature element?
- How are proof, data, cases, images, certificates, or reports framed?
- What is the motion signature?
- How does the system stay recognizable on mobile?
- What should be avoided?

## Workflow

1. Read the evidence map and site blueprint.
2. Select 2 to 3 benchmark sources or open-source templates relevant to the industry, page job, or interaction goal.
3. Extract only the design logic: grid, type scale, media treatment, section rhythm, proof module, navigation, footer, and motion.
4. Produce 2 to 3 named visual systems using the canvas above.
5. If the user wants control, ask them to choose A/B/C. If they want speed, select the safest system and state why.
6. Implement the selected system through structure and components, not just colors.
7. Screenshot-review desktop and mobile before handoff.

## Adaptation Rules From Presentation References

Presentation references can improve website taste, but they must be adapted to responsive websites.

- Chroma bands become category bands, proof trails, or case taxonomy strips.
- Paper texture becomes editorial evidence surfaces, source-backed dossiers, or report-like sections.
- Framed slide screenshots become case boards, proof ledgers, certificate walls, product snapshots, or project maps.
- Progress dots become section journey indicators, process state, or route tabs.
- Strong label/date metadata becomes source tags, status labels, version notes, and evidence boundary markers.
- Fixed 16:9 stages must not be copied directly. Websites need fluid containers, mobile breakpoints, and scroll-native rhythm.

## Named System Starters

### Editorial Evidence Chroma

- Best for: creative studios, brand strategy, public-facing skill pages, editorial professional services.
- Visual thesis: a confident editorial surface with high-saturation accents used as meaning-bearing category rails.
- Layout grammar: wide editorial intro, asymmetric proof board, color-coded evidence strips, selected cases as publication spreads.
- Signature element: chroma rail or diagonal evidence band that carries section/category identity.
- Motion: mask reveal on media, gentle band sweep, section progress indicator.
- Avoid when: the client needs conservative institutional trust, strict regulatory tone, or dense technical specifications.

### Executive Proof Dossier

- Best for: B2B service firms, consulting, agencies, operations partners, public proposal pages.
- Visual thesis: the website feels like a management-ready dossier rather than a generic service list.
- Layout grammar: source labels, ledger rows, stakeholder paths, memo panels, service maps, decision tables.
- Signature element: evidence ledger with clear source, scope, review, and action rows.
- Motion: tabs, stepper, active evidence row, restrained page transitions.
- Avoid when: the brand needs expressive consumer energy or image-led lifestyle storytelling.

### Industrial Field Blueprint

- Best for: engineering, environmental protection, equipment, manufacturing, energy, construction, site operations.
- Visual thesis: the page is a field-ready blueprint that shows process, constraints, handoff, and proof boundaries.
- Layout grammar: process line, site map, equipment/spec matrix, compliance board, project lifecycle, before/after slots.
- Signature element: technical flow map with hotspots or node states.
- Motion: SVG line draw, hotspot tour, spec tab switch, before/after comparison.
- Avoid when: there are no real site, equipment, process, or certificate assets available.

### Product Command Theater

- Best for: AI SaaS, data tools, monitoring, automation, diagnostics, platform launches.
- Visual thesis: the website stages the product as a system in motion, with clear trust boundaries and user actions.
- Layout grammar: command hero, product state sequence, workflow map, integration matrix, risk/approval module.
- Signature element: stateful command surface or dashboard sequence that changes by feature or scroll.
- Motion: panel morph, dashboard sequence, integration orbit, reduced-motion fallback.
- Avoid when: there is no product UI, no workflow, and no confirmed feature scope.

## Rejection Gate

Reject or revise the system before coding if any of these are true:

- It could be reused unchanged for an unrelated industry.
- It only changes palette, icons, or copy.
- It uses the same header, hero, cards, proof module, interaction, and footer as another demo.
- It relies on decoration while proof, evidence, service scope, or conversion path stays weak.
- Mobile collapses into generic stacked cards and loses the signature element.
- It introduces motion that does not communicate sequence, state, relationship, or user feedback.
